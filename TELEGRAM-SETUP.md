# Заявки с сайта → в Telegram

Инструкция на 15 минут. Три шага: бот → посредник → домен.

---

## Почему нельзя просто вставить токен бота в сайт

GitHub Pages — статический хостинг, сервера у него нет. Если положить токен бота прямо в код страницы, его увидит любой через «Просмотр кода страницы». С этим токеном чужой человек сможет писать от имени твоего бота и читать входящие сообщения.

Поэтому между сайтом и Telegram стоит **Cloudflare Worker** — маленький бесплатный посредник. Токен лежит у него, в код сайта не попадает.

```
Сайт  →  Cloudflare Worker  →  Telegram-бот  →  твой чат
         (токен только здесь)
```

---

## Шаг 1. Создать бота (3 минуты)

1. Открой в Telegram **@BotFather**
2. Отправь `/newbot`
3. Придумай имя: `Atmos Villas Leads`
4. Придумай логин, должен заканчиваться на `bot`: `atmos_leads_bot`
5. BotFather пришлёт **токен** — длинную строку вида
   `8123456789:AAHdqTcvCH1vGWJxfSeofSAs0K5PALDsaw`

   **Скопируй и сохрани.** Это пароль от бота, никому не показывай.

### Узнать свой chat_id

1. Открой **@userinfobot** → отправь `/start`
2. Он ответит твоим `Id`, например `512345678` — это и есть **CHAT_ID**
3. Найди своего нового бота по логину и нажми **Start** — без этого бот не сможет тебе написать

> **Если хочешь получать заявки в общий чат с командой:** добавь бота в группу, сделай его администратором, а chat_id группы возьми через @getidsbot (у групп он с минусом, например `-1001234567890`).

---

## Шаг 2. Поднять Worker (7 минут)

1. Зарегистрируйся на **https://dash.cloudflare.com** (бесплатно)
2. Слева: **Workers & Pages** → **Create** → **Create Worker**
3. Имя: `atmos-form` → **Deploy**
4. Нажми **Edit code**
5. Удали весь код в редакторе, вставь содержимое файла **`worker.js`** из этой папки
6. **Deploy** (справа сверху)

### Вписать токен и chat_id

1. Вкладка **Settings** → **Variables and Secrets** → **Add**
2. Добавь три переменные:

| Имя | Значение | Тип |
|---|---|---|
| `BOT_TOKEN` | токен из BotFather | **Secret** (зашифровать) |
| `CHAT_ID` | твой id из userinfobot | Text |
| `ALLOWED_ORIGIN` | `https://atmos.villas` | Text |

3. **Deploy** ещё раз, чтобы переменные подхватились

`ALLOWED_ORIGIN` — это защита: Worker будет принимать заявки только с твоего сайта, чтобы через него не спамили.

### Скопировать адрес Worker

Наверху страницы Worker будет ссылка вида:
```
https://atmos-form.ТВОЙ-АККАУНТ.workers.dev
```
Скопируй её.

---

## Шаг 3. Подключить к сайту (1 минута)

Открой файл **`config.js`** и вставь адрес:

```js
window.ATMOS_FORM_ENDPOINT = "https://atmos-form.ТВОЙ-АККАУНТ.workers.dev";
```

Сохрани. Всё — форма заработает.

> Пока строка пустая, форма не отправляет заявку, а показывает кнопку «Написать в WhatsApp». Сайт не ломается.

---

## Шаг 4. Залить на GitHub Pages

1. Создай репозиторий на GitHub (например `atmos-villas`)
2. Загрузи **всё содержимое этой папки** в корень репозитория
3. **Settings** → **Pages** → Source: `Deploy from a branch` → ветка `main`, папка `/ (root)` → **Save**
4. Там же **Custom domain** → впиши `atmos.villas` → **Save**
5. Поставь галочку **Enforce HTTPS** (появится через несколько минут)

### DNS у регистратора домена

Добавь записи:

| Тип | Имя | Значение |
|---|---|---|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |
| CNAME | www | `ТВОЙ-ЛОГИН.github.io` |

Домен подхватится за 10 минут – 1 час.

Файл **`CNAME`** уже лежит в папке и содержит `atmos.villas`. Если домен другой — открой его и впиши свой.
Файл **`.nojekyll`** нужен, чтобы GitHub не обрабатывал папку как блог. Не удаляй.

## Проверка

1. Открой сайт → пролистай до формы
2. Заполни: имя, телефон, способ связи → **Отправить**
3. В Telegram должно прийти:

```
🏝 New enquiry — Atmos Villas

👤 Name: John Smith
📱 Phone: +49 170 1234567
🔔 Prefers: ✈️ Telegram
🌐 Language: Deutsch
🕒 Time: 17 Jul, 14:32 (Bangkok)
🔗 https://atmos.villas/de.html

[ 💬 Reply on WhatsApp ]
```

Телефон можно нажать и скопировать одним касанием. Кнопка снизу сразу открывает переписку в WhatsApp с этим номером.

**Язык страницы приходит в заявке** — сразу видно, на каком языке отвечать.

## Если не работает

| Симптом | Причина |
|---|---|
| Форма пишет «не удалось отправить» | В `config.js` пустая строка или адрес с опечаткой |
| Worker отвечает 403 | `ALLOWED_ORIGIN` не совпадает с адресом сайта. Должно быть точно `https://atmos.villas`, без слэша в конце |
| Worker отвечает 502 | Неверный `BOT_TOKEN`, или ты не нажал **Start** у бота |
| Ничего не приходит в группу | Бот не администратор группы, либо chat_id без минуса |

Логи Worker: **Workers & Pages** → `atmos-form` → **Logs** → **Begin log stream**.

---

## Безопасность

- Токен хранится только в переменных Cloudflare, в коде сайта его нет
- `ALLOWED_ORIGIN` не даёт слать заявки с чужих сайтов
- Скрытое поле-ловушка отсеивает ботов-спамеров: если оно заполнено, Worker молча ничего не отправляет
- Имя и телефон обрезаются по длине и чистятся от служебных символов

Если токен всё же утёк — открой @BotFather → `/revoke` → выбери бота → получишь новый токен → впиши его в переменные Worker.

## Лимиты бесплатного тарифа

Cloudflare Worker: **100 000 запросов в сутки**. Для сайта с заявками это в тысячи раз больше, чем нужно. Платить не придётся.
